We would like to develop a simple, efficient, and effective method for computing $$I(x;a) = \int_{-\infty}^{\infty} e^{-a |k|} \cos k x\, \mathrm{d}k = \frac{2a}{a^{2} + x^{2}}.$$ It is hoped that by studying this problem, we develop a method useful for computing the evolution of characteristic function for the Lvy SDE.

To compute this integral numerically, it is natural to consider the finite interval, $[-L/2,L/2]$, over which we apply numerical quadrature. Hence, we rewrite $I(x;a)$ as $$I(x;a) = \int_{-L/2}^{L/2} e^{-a |k|} \cos k x\, \mathrm{d}k + \int_{L/2}^{\infty} 2 e^{-a k} \cos k x\, \mathrm{d}k.$$ One potential numerical method to approximate $I(x;a)$ would be to apply the periodic trapezoid rule (PTR) to the first integral and neglect the second integral. We know that for periodic functions the trapezoid rule is spectrally accurate provided that $N$ is sufficiently large. Doing so yields the numerical approximation, $$I(x;a) \approx \frac{L}{N} \sum_{j = 1}^{N} e^{-a |k_{j}|} \cos( k_{j} x), \quad k_{j} = -\frac{L}{2} + (j-1) \frac{L}{N}, \quad j = 1, \cdots, N.$$ This numerical method can be implemented efficiently using Fast Fourier transforms.

There are two main causes for error in using this numerical method.

  • Aliasing -- We incur an error if $N$ is not sufficiently large that it resolves the oscillations of the integrand, which becomes especially problematic for large values of $x$.

  • Truncation -- We incur an error by neglecting the integral over $[L/2,\infty)$, which becomes especially problematic for small values of $a$.

If we would like to evaluate $I(x;a)$ for $x \gg 1$ and $a \ll 1$, both causes for error will be significant. It is for this reason that we consider a different numerical method.

Addressing aliasing error¶

Aliasing error is caused by the oscillatory behavior of the integrand. When the frequency of oscillations is higher than the mesh width of the quadrature rule, the quadrature rule is unable to accurately integrate that function. A simple solution would be to refine the mesh so that the highest frequency is resolved on it.

For a fixed value of $x$, there exists an $L$ such that the finite interval will yield an accurate approximation to the integral. Given that value of $L$, one can then determine an appropriate $N$. However, if $x$ is to vary over a large range of values, then larger values of it will require larger values of $L$ and $N$ to achieve a uniform accuracy.

If we relax the requirement for spectral accuracy, we can mitigate this issue using a so-called product Nystrm quadrature rule. Consider the integral $$ I = \int_{k_{j}}^{k_{j+1}} f(k) \cos k x\, \mathrm{d}k.$$ Suppose that $f(x)$ is smooth over $[k_{j},k_{j+1}]$ so that its first-order Lagrange interpolant, $$\ell_{1}(k) = \frac{k - k_{j}}{\Delta k} f(k_{j} + \Delta k) - \frac{k - k_{j+1}}{\Delta k} f(k_{j}),$$ is a good approximation. We replace $f(k)$ by $\ell_{1}(k)$ and find $$ I \approx \int_{k_{j}}^{k_{j+1}} \ell_{1}(k) \cos k x \mathrm{d}k = \frac{m_{1}(k_{j}) - k_{j} m_{0}(k_{j})}{\Delta k} f(k_{j+1}) - \frac{m_{1}(k_{j}) - k_{j+1} m_{0}(k_{j})}{\Delta k} f(k_{j}), $$ with $$ m_{0}(k_{j}) = \int_{k_{j}}^{k_{j+1}} \cos k x\, \mathrm{d}k = \begin{cases} \displaystyle \frac{\sin(k_{j+1} x) - \sin(k_{j} x)}{x} & x \neq 0,\\ k_{j+1} - k_{j} & x = 0, \end{cases} $$ and $$ m_{1}(k_{j}) = \int_{k_{j}}^{k_{j+1}} k \cos k x\, \mathrm{d}k = \begin{cases} \displaystyle \frac{\cos(k_{j+1} x) - \cos( k_{j} x) + x [ k_{j+1} \sin( k_{j+1} x) - k_{j} \sin( k_{j} x ) ]}{x^{2}} & x \neq 0,\\ \frac{1}{2} \left[ k_{j+1}^{2} - k_{j}^{2} \right] & x = 0. \end{cases} $$ For convenience, we introduce $$ w_{j} = - \frac{m_{1}(k_{j}) - k_{j+1} m_{0}(k_{j})}{\Delta k},$$ and $$ w_{j+1} = \frac{m_{1}(k_{j}) - k_{j} m_{0}(k_{j})}{\Delta k},$$ so that this product Nystrm quadrature rule can be written compactly as $$ I = \int_{k_{j}}^{k_{j+1}} f(k) \cos k x\, \mathrm{d}k \approx w_{j} f(k_{j}) + w_{j+1} f(k_{j+1}).$$

This quadrature is effective at addressing aliasing because it integrates the oscillatory part of the integrand that depends on the value of $x$ exactly. The only approximation made is replacing $f(k)$ by its first-order Lagrange interpolant. In principle, we can obtain a higher-order quadrature rule by replacing $f(k)$ by a higher-order Lagrange interpolant.

We now test this numerical method. Consider the integral $$ \int_{-L/2}^{L/2} e^{- a k } \cos k x\, \mathrm{d}k = 2 \frac{a - e^{-a L/2} \left[ a \cos( x L/2 ) - x \sin( x L / 2 ) \right]}{a^{2} + x^{2}}. $$ We compute this integral numerically using the periodic trapezoid rule and the product quadrature rule given above.

# compare the periodic trapezoid rule with the product quadrature rule

from math  import exp, cos, sin, pi

import numpy as np
import matplotlib.pyplot as plt

# set the value of a

a = 0.01

# compute the k grid for integration

Nk = 128
L  = 12.8
dk = L / Nk
k  = np.arange( -L/2, L/2, dk )

# compute set of x values

Nx = 101
x  = np.linspace( 0, 1000, Nx )

# allocate memory for the results

exact_finite   = 0 * x # store the exact result
approx1_finite = 0 * x # store the result of the periodic trapezoid rule
approx2_finite = 0 * x # store the result of the product quadrature rule

# loop over all values of x

for i in range( 0, Nx ):

    # initialize the value of integrals

    trapz = 0
    prodq = 0

    # compute the product quadrature rule

    for j in range( 0, Nk ):
   
        # compute the quadrature points
    
        k1 = k[j]
        k2 = k[j] + dk
        
        # -----------------------
        # PERIODIC TRAPEZOID RULE
        # -----------------------
        
        f1 = exp( -a * abs(k1) ) * cos( k1 * x[i] )
        f2 = exp( -a * abs(k2) ) * cos( k2 * x[i] )
        
        trapz += 0.5 * dk * ( f2 + f1 )
        
        # -----------------
        # PRODUCT QUARATURE
        # -----------------
        
        # compute the moments
        
        if x[i] == 0:
            
            m0 = k2 - k1
            m1 = 0.5 * k2**2 - 0.5 * k1**2
            
        else:
            
            m0 = sin( k2 * x[i] ) / x[i] - sin( k1 * x[i] ) / x[i]
            
            m1 = ( cos( k2 * x[i] ) / x[i]**2 - cos( k1 * x[i] ) / x[i]**2
            + k2 * sin( k2 * x[i] ) / x[i] - k1 * sin( k1 * x[i] ) / x[i] )
            
        # compute the quadrature weights

        w1 = - ( m1 - k2 * m0 ) / dk
        w2 =   ( m1 - k1 * m0 ) / dk
    
        # compute the function values at the quadrature points
   
        f1 = exp( -a * abs(k1) )
        f2 = exp( -a * abs(k2) )
   
        # compute the trapezoid rule

        prodq += w1 * f1 + w2 * f2
    
    # store the results
    
    approx1_finite[i] = trapz
    approx2_finite[i] = prodq
    
    # --------------    
    # EXACT SOLUTION
    # --------------
    
    exact_finite[i] = ( 2 * exp( - a * L / 2 ) / ( a**2 + x[i]**2 ) 
                * ( a * exp( a * L / 2 ) - a * cos( L * x[i] / 2 ) + x[i] * sin( L * x[i] / 2 ) ) )


# plot the results

plt.semilogy( x, abs( approx1_finite - exact_finite ), '-', label = 'periodic trapezoid rule' )
plt.semilogy( x, abs( approx2_finite - exact_finite ), '-', label = 'product quadrature rule' )
plt.grid( True )
plt.xlabel( 'x' )
plt.ylabel( 'error' )
plt.title( 'Error results for a finite interval')
legend = plt.legend( loc='center right' )

These results show that the product quadrature rule is capable of computing the integral accurately over a very large range of $x$ values with small error. In contrast, the periodic trapezoid rule exhibits an error between $10^{-2}$ and $1$ for moderate-to-large values of $x$.

Addressing truncation error¶

We now address truncation error. For the integral defining $I(x;a)$ above, we would like $$ \int_{L/2}^{\infty} e^{-a k} \cos k x\, \mathrm{d}k \ll 1,$$ so that $$I(x;a) \approx \int_{-L/2}^{L/2} e^{-a |k|} \cos k x\, \mathrm{d}k,$$ serves as a good approximation. That truncation will be a good approximation as long as $a L$ is sufficiently large that $e^{-a L/2}$ is negligibly small. However, when $a \ll 1$, this restriction requires that $L$ be increased to obtain a truncation error that is uniform for larger values of $a$. Alternatively, we can determine the asymptotic approximation for this integral in the limit as $a \to 0^{+}$, and use that to correct this truncation error.

For this problem, we can compute this integral exactly and obtain the result, $$ \int_{L/2}^{\infty} 2 e^{-a k} \cos k x\, \mathrm{d}k = 2 e^{-a L/2} \frac{a \cos( x L / 2 ) - x \sin( x L / 2 )}{a^{2} + x^{2}}. $$ We add this exact integral to the periodic trapezoid rule and product quadrature rule approximations over the finite intervals computed above to see how well they approximate $I(x;a)$.

# compute the truncated portion of I(x;a)

truncation = 2 * exp( -a * L / 2 ) * ( a * np.cos( x * L / 2 ) - x * np.sin( x * L / 2 ) ) / ( a**2 + x**2 )

# add this truncated portion to the approximations over the finite interval

approx1 = approx1_finite + truncation
approx2 = approx2_finite + truncation

# compute the exact result

exactresult = 2 * a / ( a**2 + x**2 )

# plot the error results

plt.semilogy( x, abs( approx1_finite - exactresult ), '-', label = 'periodic trapezoid rule (truncated)' )
plt.semilogy( x, abs( approx1        - exactresult ), '--', label = 'periodic trapezoid rule (corrected)' )
plt.semilogy( x, abs( approx2_finite - exactresult ), '-', label = 'product quadrature rule (truncated)' )
plt.semilogy( x, abs( approx2        - exactresult ), '--', label = 'product quadrature rule (corrected)' )
plt.ylim( 1e-15, 1e8 )
plt.grid( True )
plt.xlabel( 'x' )
plt.ylabel( 'error' )
plt.title( 'Error results over the whole line' )
legend = plt.legend( loc = 'upper right' )

These results show that adding the truncation is important for obtaining an accurate result. The truncated approximations are shown as solid curves above. Those results are limited by the truncation error. By adding the truncation to the periodic trapezoid rule, we still obtain a substantial error, presumably due to aliasing. However, upon adding the truncation to the product quadrature rule, we obtain a much smaller error.

Even though the results above look quite promising, we would like to consider a general method to approximate this integral for problems when this integral cannot be computed exactly. For this integral, $e^{-a k}$ is monotone decreasing over $[L/2,\infty)$, so we obtain the asymptotic expansion through repeated integration-by-parts. Upon two applications of integration-by-parts, we find that $$ \int_{L/2}^{\infty} e^{-a k} \cos k x\, \mathrm{d}k = - \frac{1}{x} e^{-a L/2} \sin( x L / 2) + \frac{a}{x^{2}} e^{-a L/2} \cos( x L / 2) - \frac{a^{2}}{x^{2}} \int_{L/2}^{\infty} e^{-a k} \cos k x\, \mathrm{d}k, $$ from which we find by recursion, $$ \begin{aligned} \int_{L/2}^{\infty} e^{-a k} \cos k x\, \mathrm{d}k &= - \frac{1}{x} e^{-a L/2} \sin( x L / 2) + \frac{a}{x^{2}} e^{-a L/2} \cos( x L / 2) - \frac{a^{2}}{x^{2}} \left[ - \frac{1}{x} e^{-a L/2} \sin( x L / 2) + \frac{a}{x^{2}} e^{-a L/2} \cos( x L / 2) - \frac{a^{2}}{x^{2}} \int_{L/2}^{\infty} e^{-a k} \cos k x\, \mathrm{d}k \right],\\ &= - \frac{1}{x} e^{-a L/2} \sin( x L / 2) + \frac{a}{x^{2}} e^{-a L/2} \cos( x L / 2) + \frac{a^{2}}{x^{3}} e^{-a L/2} \sin( x L / 2) - \frac{a^{3}}{x^{4}} e^{-a L/2} \cos( x L / 2) + \frac{a^{4}}{x^{4}} \int_{L/2}^{\infty} e^{-a k} \cos k x\, \mathrm{d}k,\\ &= - \frac{1}{x} \left[ 1 - \frac{a^{2}}{x^{2}} + \cdots \right] e^{-a L/2} \sin( x L / 2) + \frac{a}{x^{2}} \left[ 1 - \frac{a^{2}}{x^{2}} + \cdots \right] e^{-a L/2} \cos( x L / 2) \end{aligned} $$ Next, we use this asymptotic approximation instead of the exact truncation result to correct the periodic trapezoid rule and product quadrature rule.

# compute the asymptotic approximation for the truncation

truncation_approx     = 0 * x
truncation_approx[0]  = 2 * exp( - a * L / 2 ) / a
truncation_approx[1:] = 2 * ( exp( -a * L / 2 ) / x[1:] 
                             * ( a / x[1:] * np.cos( x[1:] * L / 2 ) - np.sin( x[1:] * L / 2 ) ) 
                             * ( 1 - a**2 / x[1:]**2 ) )

# add this approximate truncation to the product quadrature rule results over the finite interval

approx2a = approx2_finite + truncation_approx
